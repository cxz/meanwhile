What is Meanwhile? back

Meanwhile is an open source project for connecting as a client to a Lotus Sametime Connect community. There is a stand-alone library portion, which can be used to write a client, and there is a Gaim protocol plugin which turns Gaim into a Sametime client. There is also a significant amount of work into documenting the Sametime protocol.

Is this project affiliated with IBM or Lotus? back

No. The Meanwhile project is neither affiliated with, nor endorsed or supported by, either IBM or Lotus.

The project was first available exclusively inside of IBM, due to questions of code ownership. These have been resolved, and Meanwhile is now open to the world.

There is still a residual internal project-- with some useful IBM-specific features added on-- available from the IBM Intranet at http://w3.opensource.ibm.com/~meanwhile (note: IBM-only link)

On what platforms can I use Meanwhile? back

Older versions of Meanwhile has been successfully compiled and used on Linux, Win32 (via mingw), and Darwin (Mac OSX). Possibly and probably more, but those are the only ones I've heard of so far.

Why is there no SRPM? back

Primarily because you can make a source-rpm from the release tarball with a single command:

rpmbuild -ts [tarball]

What is Gaim? back

Gaim is an open source Instant Messaging client platform. Gaim supports almost every instant messaging protocol, and any number of additional protocols can be added in as modules. It provides a single front end for all of a user's various instant messaging accounts and buddy lists.
I have the source, now how do I compile this? back

Ah, now that's tricky.

This plugin is dynamically linked against libMeanwhile, which must already be installed. That part at least, is easy. Just make sure that you've run ldconfig, and detection should work find.

This is also a Gaim plugin, which naturally requires headers from Gaim. However, Gaim has no real development package, so the only way to get the Gaim headers is to get the Gaim source. Once you've obtained that, running the following should produce Makefiles which build the plugin.

./configure --with-gaim-source=$path_to_gaim/src

This is outlined in the INSTALL file.
Why is the SRPM not building the Gaim plugin? back

The reasoning is the same as outlined above. The fix for this is to rebuild the rpm with the environment variable gaim_src_dir set to the absolute path to the src directory for Gaim. For example:

gaim_src_dir=/path/to/gaim/src rpmbuild --rebuild meanwhile-0.2-1.srpm

Why aren't some of my buddies showing as online? back

It's been noted that the sametime protocol seems to be case-sensitive in the identification of its users, unlike virtually every other internet identity. Even unlike virtually every use of your IBM Intranet User ID.

Therefore, if your buddy has been entered as 'mybuddy', when in fact their ID is 'MyBuddy', sametime will happily accept the addition into your buddy list but never provide any status updates. You will also encounter confusing problems if they message you, as gaim will attempt to reply to the corresponding ID in your buddy-list, and sametime's messaging service will erroneously inform you that the user you're sending to is not online.
